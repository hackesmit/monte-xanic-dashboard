# Wave 1 #3 — Calidad badge in mediciones table + live score in edit modal

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Each row in the Mediciones Técnicas table shows an A+/A/B/C grade badge with score, and the edit modal shows a live-updating grade that re-scores as the user edits the form. Lab can see at a glance which lots are passing the rubric.

**Architecture:** `scoreLot()` expects berry chemistry on the top-level lot object plus a snake-cased nested `medicion`. Mediciones rows have neither. Add a new pure helper `scoreFromMedicion(medicionRow, berryByLot)` in `js/classification.js` that resolves the matching berry by `(lotCode, vintage)` and delegates to `scoreLot`. `js/mediciones.js` builds the berry index once per `refresh()` and calls the helper per row.

**Tech Stack:** Vanilla JS ES modules, node:test for unit tests, vanilla HTML5 `<dialog>` for the modal.

**Spec:** `docs/superpowers/specs/2026-05-29-wave1-dispatch-design.md` § Subagent D.

**Dependency:** This plan's browser smoke depends on Wave 1 #2 having merged first (so demo-mode mediciones can find matching berries). The code itself is independent. Run `git log main --oneline` before starting and confirm the #2 commit (`fix(demo): emit current-vintage mediciones + receptions for calidad map`) is present.

```javascript
// ── scoreFromMedicion: lift a medicion row into the lot shape scoreLot expects ──
// Mediciones rows lack berry chemistry (brix, pH, tANT, …). To grade a medicion,
// we look up its matching berry by (lotCode, vintage), graft the medicion's
// snake-cased health fields onto a clone, and delegate to scoreLot.
//
// Mirrors what joinBerryWithMediciones in dataLoader.js does for the map view,
// but starts from the medicion side so the Mediciones Técnicas table can show
// a per-row badge without requiring the user to navigate to the berry view.
//
// @param {object} m — mediciones_tecnicas row (camelCase from _rowToMedicion)
// @param {Map<string, object>|null} berryByLot — keyed `${lotCode}||${vintage}`
// @returns {object} same shape as scoreLot, or { grade: null, reason: 'Sin berry' }
export function scoreFromMedicion(m, berryByLot) {
  if (!m || !m.lotCode || m.vintage == null) {
    return { grade: null, score36: null, rubricId: null, missing: [], reason: 'Sin berry' };
  }
  if (!berryByLot || typeof berryByLot.get !== 'function') {
    return { grade: null, score36: null, rubricId: null, missing: [], reason: 'Sin berry' };
  }
  const berry = berryByLot.get(`${m.lotCode}||${m.vintage}`);
  if (!berry) {
    return { grade: null, score36: null, rubricId: null, missing: [], reason: 'Sin berry' };
  }
  const lot = {
    ...berry,
    medicion: {
      health_grade:       m.healthGrade,
      health_madura:      m.healthMadura,
      health_inmadura:    m.healthInmadura,
      health_sobremadura: m.healthSobremadura,
      health_picadura:    m.healthPicadura,
      health_enfermedad:  m.healthEnfermedad,
      health_quemadura:   m.healthQuemadura,
      tons_received:      m.tons,
      phenolic_maturity:  m.phenolicMaturity
    }
  };
  return scoreLot(lot);
}
```

```javascript
  _updateLiveScore() {
    const el = document.getElementById('med-edit-score');
    if (!el) return;
    // Build a synthetic medicion from the current form values so the live
    // grade reflects unsaved edits. Then re-run scoreFromMedicion using the
    // berry index captured during refresh().
    const fv = (id) => document.getElementById(id)?.value;
    const fp = (id) => {
      const v = fv(id);
      return v == null || v === '' ? null : parseFloat(v);
    };
    const editing = this._editingRow || {};
    const synthetic = {
      lotCode: editing.lotCode,
      vintage: editing.vintage,
      variety: editing.variety,
      appellation: editing.appellation,
      tons: fp('med-edit-tons') ?? editing.tons,
      healthGrade: fv('med-edit-grade') ?? editing.healthGrade,
      healthMadura: fp('med-edit-health-madura') ?? editing.healthMadura,
      healthInmadura: fp('med-edit-health-inmadura') ?? editing.healthInmadura,
      healthSobremadura: fp('med-edit-health-sobremadura') ?? editing.healthSobremadura,
      healthPicadura: fp('med-edit-health-picadura') ?? editing.healthPicadura,
      healthEnfermedad: fp('med-edit-health-enfermedad') ?? editing.healthEnfermedad,
      healthQuemadura: fp('med-edit-health-quemadura') ?? editing.healthQuemadura,
      phenolicMaturity: fv('med-edit-phenolic-maturity') ?? editing.phenolicMaturity
    };
    const score = scoreFromMedicion(synthetic, this._berryByLot);
    el.innerHTML = this._renderGradeBadge(score);
  },
```
